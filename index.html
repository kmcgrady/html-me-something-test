Step One - The very first step in creating an HTML file

Like with many aspects of coding, creating an HTML page requires the use of text editor. Go ahead and fire up your favorite. While many people in class like to use Visual Studio Code, I prefer to use the open source program Atom.

Look at that, a nice blank page. This could be used for any language, really, but we will be writing good old HTML!

Step 2 - Really getting started

What use is a text editor if there's no text? To get us started, Atom ca be pretty useful. By simply typing html into the editor, the boilerplate of an HTML page is automatically generated. This can be particularly useful for beginers, as it shows the basic layout of an HTML page as well as the head and doctype elements.

There we go, one fresh set of boilerplate. Now we are ready to move on to the next step...

Step 3 - Even more HTML!

The boilerplave that we generated is a good starting point, but it certainly isn't the end goal. If we were to open our browser to the page we just generated, with nothing but the boilerplate, we would see a completley blank page. To put out content in, we need to use tags. Tags are the snippets of code that include and are between angle brackets, like <html>,<body>, and <head>. Below we can see an HTML page that includes a lot of different tags for different sorts of content. These new tags include the unordered list tag <ul>, the heading tags<h1> and <h2>, and the paragraph tag <p>. All of these tags combine together to express content in different ways, for different purposes. For an overview of many of the different HTML tags available, please see W3Schools' HTML Tag List.

Also notice that some of the code is surrounded by the comment markers <!-- and -->. Any code or text that is between these markers are not iterpreted by the browser, which can make it very useful when trying new code or bug fixing to isolate code.

Step 4 - Checking Your Work

Okay, now let's check and see what our page looks like in the browser:

Looks like all the content we included is shown on the page, but it doesn't look very appealing. That will change when we get a chance to look at Step 5.

Step 5 - Adding Some Style

Okay, now that we have our content written and showing up correcly in the browser, we can go ahead and adjust how that content looks. We do that by employing CSS code into our page. To do this, we need to add the <link> tag to the head of the document. For example, in this page the <link> tag points to an external CSS file called styles.css. To do that, we expand the attributes of the <link> tag so it looks like this:

<link rel='stylesheet' type='text/css' href='styles.css' />

From there, we can then begin coding our new file, styles.css. You can get a good overview of what sort of thins CSS can do by looking at W3Schools' CSS Reference. See below what the CSS looks like in the Atom text editor.

Just looking at the code in the text editor, it can be a bit confusing to see if the code is doing what you inteded. That's why it is important to constanly be checking what the page looks like in a browser window. You will see below how this example page looks when rendered by the browser, now that the page has CSS.

There we go! Looks pretty good. At least, quite a bit better than it did without CSS. Now that we have our HTML and CSS done, we can go ahead and get to the last and most involved step: Step 6.

Step 6 - Revise, Revise, and Revise Again

This is the step in the development process that always takes the longest, and never really seems to be complete. Go ahead and go into your index.html file and catch any typos, badly worded phrases, or make changes as you see fit. The end of the project is when you decide it it.

Likewise, don't be afraid to go into styles.css and make changes. Perhaps the background-color you picked for the <h3> tag should be green instead of orange. Perhaps the padding-top of the <p> tags in the magic-bean class should be 15px rather than 25px. There are so many different ways that your code can be displayed, and there is no one right way to do it.

Once you've tweaked your code to a point that you are happy with it, go ahead and submit it and give yourself a pat on the back. You earned it!
